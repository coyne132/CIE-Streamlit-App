CIE Navigation Center Siting DashboardDecision Support Workflow & MethodologyPurpose of the ToolThis dashboard is designed to support prioritization of candidate areas for Community Information Exchange (CIE) Navigation Center siting within Franklin County.The tool:Identifies census tracts with elevated structural barriers to accessSupports scenario‑based prioritization aligned with policy goalsSurfaces existing Digital Skills Hub (DSH) partner locations within priority areasThe dashboard does not determine final site selection. All outputs are intended to inform, not replace, human decision‑making.Unit of Analysis and ActionAnalytical unit: Census tracts(used to measure and compare structural access barriers)Action unit: Partner locations(DSH sites located within prioritized census tracts)Census tracts provide a consistent, population‑based geography for analysis, while partner locations represent deployable infrastructure.Data SourcesAmerican Community Survey (ACS) 2020–2024, 5‑Year EstimatesFranklin County census tract geometries (2024)Digital Skills Hub partner location dataPublic transportation stop locations (COTA)Stakeholder survey input for weighting (used in hybrid score construction)Public Library dataZCTA data used for geographic contextStructural Access Disparity Index (Base Score)4.1 Barrier CategoriesStructural access barriers are grouped into seven categories:CommunicationEconomicTransportationDigitalEducationFunctionalHealth AccessEach category represents a distinct dimension of access limitation.4.2 Category Score ConstructionIndividual ACS indicators are standardized using z‑score normalizationIndicators within each category are averaged to form category‑level scoresThis prevents any single indicator from dominating the analysis4.3 PCA‑Based Composite ScorePrincipal Component Analysis (PCA) is applied to the category scoresThe first principal component (PC1) is retainedPC1 captures the dominant, shared pattern of structural access disparityAll category loadings on PC1 are positive and similar in magnitudeThe result is a raw composite barrier score for each census tract.4.4 NormalizationThe composite barrier score is min–max normalized to a 0–1 scale for interpretability:Higher values indicate greater structural access barriersLower values indicate fewer barriers, relative to other tractsThis normalized value is referred to as the Structural Access Score.Population Eligibility FilterTo prevent tracts with extremely small populations from disproportionately influencing results:Census tracts in the bottom 10% of total population are excludedRemaining tracts are considered eligible for ranking and prioritizationThis filter is applied before any ranking or prioritization logic.Base Ranking (Structural Need Only)Eligible census tracts are ranked in descending order by their Structural Access Score.This produces a baseline ranking that reflects structural need only, without policy modifiers or feasibility constraints.Optional Prioritization (Boosting Logic)7.1 Purpose of BoostingPrioritization options allow decision‑makers to emphasize specific populations or policy goals, while preserving the underlying structural access assessment.Boosting:Affects ranking orderDoes not change the Structural Access Score itself7.2 Prioritization VariablesExamples include:Households with childrenUninsured youthEarly childhood povertyAging populationsVeteran communitiesPresence of existing DSH partners7.3 Boosting MethodA new column, Adjusted Priority Score, is createdThe Structural Access Score is multiplied by small priority modifiers(10%)Modifiers are applied only when corresponding prioritization options are selectedTracts are re‑ranked using the adjusted score.Anti‑Clustering (Geographic Coverage Constraint)8.1 PurposeAnti‑clustering promotes broader geographic coverage by preventing multiple prioritized tracts from being directly adjacent.This supports:Equitable service distributionAvoidance of redundant site placement8.2 How Anti‑Clustering WorksTracts are sorted by priority rankTracts are selected iterativelyA tract is included only if its centroid is at least a minimum distance from previously selected tractsSelection continues until the Top‑N target is reachedAnti‑clustering:Does not modify scoresOnly affects which tracts are selected into the Top‑N8.3 User ControlAnti‑clustering is optionalEnabled through an “Advanced Prioritization” settingMinimum distance is fixed or chosen from a small, predefined setFinal Selection of Priority AreasAfter applying:Population eligibility filteringOptional prioritization boostsOptional anti‑clustering constraintsThe dashboard selects the final Top‑N priority census tracts.These tracts represent high‑need areas under explicit, user‑selected assumptions.Barrier Category BreakdownTo ensure interpretability and transparency:Each prioritized tract includes a barrier category breakdownCategory contributions are expressed as proportional influences on the composite scoreAn aggregate breakdown is also shown across all Top‑N tracts to reveal systemic patternsThis explains why areas rank highly, not just where they rank.Partner Locations in Priority AreasFor each Top‑N census tract:All intersecting Digital Skills Hub partner locations are identifiedResults are presented as:Partner nameLocation nameStreet addressAssociated priority rankThis bridges analytical prioritization with real‑world deployment options.Map Visualization (Choropleth)The dashboard includes an interactive map that:Displays all census tracts colored by Structural Access ScoreHighlights Top‑N priority tractsOverlays partner locations as point markersThe map provides spatial context and supports discussion of coverage, proximity, and feasibility.Scenario Awareness and GovernanceEach dashboard view clearly displays:The active prioritization scenarioWhether anti‑clustering is enabledUsers can reset the dashboard to the default structural‑need‑only scenario at any time.This ensures transparency and reproducibility in decision‑making contexts.Export and DocumentationUsers may export:Priority area rankingsPartner locations in prioritized areasActive assumptions and scenario settingsExports support:Meeting documentationAuditabilityFollow‑up planning workflowsIntended Use and LimitationsThis dashboard supports strategic prioritization and scenario exploration.It does not:Determine exact facility locationsAccount for site‑specific constraints (zoning, leases, hours, staffing)Replace local knowledge or stakeholder inputFinal siting decisions should incorporate additional operational, community, and political considerations.
